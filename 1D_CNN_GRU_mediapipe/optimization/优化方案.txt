🚀 1. 核心感知引擎：从“沉重缝合怪”到“轻量双引擎”
你的旧版： 使用了 HolisticLandmarker。这玩意是个黑盒，即使你只取 75 个点，它在底层依然会强行计算 468 个面部网格点，极大地拖慢了速度，并且容易触发 Linux 下的 Batch Size Bug。

优化后版： 彻底抛弃 Holistic，使用了 PoseLandmarker (Lite) + HandLandmarker 双引擎并发。砍掉了 100% 的面部冗余计算，只提取你真正需要的身体和手部点。

⚡ 2. 追踪模式：从“逐帧盲猜”到“时序追踪”
你的旧版： 使用了 RunningMode.IMAGE。这意味着 150 帧图像，MediaPipe 会把它当成 150 张毫不相干的照片，每一帧都进行一次极其耗时的“冷启动全图搜索”。

优化后版： 切换到了 RunningMode.VIDEO，并引入了 global_ts_ms（严格递增的全局时间戳）。这让 MediaPipe 开启了卡尔曼滤波追踪，第一帧找到人之后，后续 149 帧直接在局部区域微调，速度直接翻倍。

🧠 3. 硬件加速：打通 4090 的任督二脉
你的旧版： BaseOptions 里没有强制指定 Delegate。在 Linux 下，这通常会导致 MediaPipe 默默退回到 CPU（使用 XNNPACK）去跑卷积网络，完全浪费了 RTX 4090。

优化后版： 显式注入了 delegate=python.BaseOptions.Delegate.GPU，强行把视觉特征提取的计算图推给了 4090 的 CUDA/EGL 核心。

📉 4. 显存带宽优化：图像预处理缩放
你的旧版： 直接把原始 frame（可能是 1080P）丢给 MediaPipe。高清图像在 CPU 到 GPU 显存之间拷贝纹理时，会造成严重的 I/O 拥堵。

优化后版： 增加了一行 cv2.resize(frame, (480, 480))。这极大地减轻了 GPU 的显存带宽压力，对于特征点检测来说，480p 的精度已经完全溢出够用了。

📊 5. 数据可视化：从“三张散图”到“专业堆叠图”
你的旧版： 保存了三张独立的柱状图（MediaPipe耗时、推理耗时、总耗时），在排版和对比时非常不直观。

优化后版： 将其合并为一张专业的 堆叠柱状图 (Stacked Bar Chart)。在一个柱子里，下半部分（绿色）是感知耗时，上半部分（蓝色）是小脑决策耗时，总链路瓶颈一目了然。同时，报告文本也严格按照你要求的 Markdown 表格格式输出。
